# Todo App

## Resumen

Todo App es una aplicación de gestión de tareas basada en el navegador desarrollada con JavaScript Vanilla para explorar las capacidades modernas de los navegadores sin depender de frameworks frontend.

Aunque está inspirada en la clásica aplicación de tareas, esta implementación amplía el concepto mediante almacenamiento persistente local, organización de tareas mediante arrastrar y soltar y temas visuales personalizables, ofreciendo una experiencia más completa de aplicación ejecutada íntegramente en el cliente.

---

## Motivación

El objetivo principal de este proyecto fue comprender mejor cómo las aplicaciones web modernas gestionan su estado completamente dentro del navegador.

En lugar de limitarse a operaciones CRUD básicas, el proyecto explora varias funcionalidades nativas del navegador presentes en aplicaciones reales.

Los principales objetivos de aprendizaje incluyeron:

- Persistencia en el navegador.
- APIs Web.
- Interacciones drag and drop.
- Gestión de estado en el cliente.
- Interfaces responsivas.
- Optimización del rendimiento.

---

## Implementación Técnica

La aplicación fue desarrollada utilizando:

- JavaScript Vanilla
- HTML
- CSS

Las principales funcionalidades incluyen:

- Creación y eliminación de tareas.
- Seguimiento de tareas completadas.
- Reordenamiento mediante drag and drop.
- Persistencia con Local Storage.
- Cambio de temas visuales.
- Interfaz responsiva.

El estado de la aplicación se gestiona completamente en el cliente mientras que los datos del usuario se almacenan localmente utilizando la API Local Storage del navegador.

Algunos de los desafíos abordados durante el desarrollo fueron:

# Preguntas Frecuentes

## ¿Qué es Todo App?

Es una aplicación de gestión de tareas basada en el navegador desarrollada con JavaScript Vanilla para explorar APIs web y arquitectura de aplicaciones del lado del cliente.

## ¿Qué tecnologías utiliza?

La aplicación está desarrollada con HTML, CSS y JavaScript Vanilla, integrando además APIs nativas del navegador como Local Storage y Drag and Drop.

## ¿Por qué construir una aplicación de tareas?

Las aplicaciones de tareas son un excelente entorno para aprender gestión de estado, persistencia en navegador, manejo de eventos y arquitectura de interfaces sin introducir complejidad innecesaria.

## ¿La aplicación almacena datos en línea?

No.

Las tareas se almacenan localmente utilizando la API Local Storage, permitiendo conservar la información entre sesiones sin necesidad de un servicio backend.

## ¿Cuál fue el principal objetivo de aprendizaje?

Comprender APIs del navegador, gestión de estado, actualizaciones de interfaz e interacción con el usuario mientras se desarrolla una aplicación mantenible sin frameworks externos.

## ¿A quién está dirigido este proyecto?

A desarrolladores interesados en fundamentos de JavaScript, APIs web, arquitectura frontend y desarrollo de aplicaciones ejecutadas completamente en el navegador.
