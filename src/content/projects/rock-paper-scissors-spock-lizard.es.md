# Piedra Papel Tijeras Lagarto Spock

## Descripción General

Piedra Papel Tijeras Lagarto Spock es una implementación para navegador de la popular variante de cinco opciones del juego clásico. El proyecto fue desarrollado durante una competencia de programación como una oportunidad para explorar arquitectura frontend, pipelines de renderizado y gestión de estado utilizando únicamente JavaScript Vanilla.

La interfaz de usuario está basada en el diseño original de Frontend Mentor, mientras que toda la lógica del juego fue implementada desde cero.

---

## Motivación

Aunque parece simple a primera vista, los juegos interactivos son un excelente entorno para aprender conceptos fundamentales de ingeniería de software.

Este proyecto se centra en:

- Gestión de estado del juego.
- Actualizaciones de renderizado.
- Interacción con el usuario.
- Lógica determinista.
- Organización de componentes sin frameworks frontend.

Desarrollar la aplicación con JavaScript puro permitió comprender con mayor profundidad cómo funcionan internamente las aplicaciones web.

---

## Implementación Técnica

La aplicación fue desarrollada utilizando:

- JavaScript Vanilla
- HTML
- CSS

Entre sus principales características se encuentran:

- Jugabilidad interactiva.
- Motor de decisión con cinco reglas.
- Sistema de puntuación.
- Interfaz responsiva.
- Transiciones animadas entre estados.
- Lógica modular del juego.

El algoritmo de decisión está separado intencionalmente de la capa de presentación, facilitando la comprensión y extensión de las reglas.

# Preguntas Frecuentes

## ¿Qué es este proyecto?

Es una implementación web de Piedra Papel Tijeras Lagarto Spock desarrollada con JavaScript Vanilla durante una competencia de programación.

## ¿Qué tecnologías utiliza?

La aplicación fue desarrollada con HTML, CSS y JavaScript Vanilla sin utilizar frameworks frontend.

## ¿Cuál era el objetivo de aprendizaje?

El proyecto fue creado para comprender mejor los pipelines de renderizado, la gestión de estado y la arquitectura frontend mediante una aplicación interactiva práctica.

## ¿Por qué no usar React o Vue?

El objetivo era comprender los fundamentos detrás de los frameworks frontend modernos implementando el comportamiento de la aplicación directamente con APIs del navegador y JavaScript.

## ¿Dónde está implementada la lógica del juego?

Las reglas están separadas de la capa de presentación, lo que facilita entender, mantener y extender la implementación.

## ¿A quién está dirigido este proyecto?

A desarrolladores interesados en ingeniería frontend, arquitectura de aplicaciones web y en aprender cómo construir interfaces interactivas sin depender de frameworks externos.
